1. OSI,TCP/IP的分层结构与功能

体系结构

OSI体系结构分为7层，从上到下依次为：应用层，表示层，回话层，传输层，网络层，数据链路层，物理层。

TCP/IP体系结构分为：应用层，运输层，网际层，网络接口层。

五层的体系结构：应用层，运输层，网络层，数据链路层，物理层。

各层的功能

应用层

应用层的功能是通过应用程序之间的交互来完成特定的网络应用。应用层协议定义的是指应用程序间通信和交互的规则。对于不同的应用程序有着不同的应用协议。比如DNS，FTP，TELNET等等

表示层

tcp/ip第五版描述的是：表示层的功能是完成数据格式的转变，应用程序和网络通信中数据格式的转变。

会话层

会话的建立，管理和终止。

传输层

传输层的主要功能为两台主机之间的进程之间的通信提供通用的数据传输服务。

网络层

网络层的主要功能为寻找，路由选择，分组转发

数据链路层

数据链路层的主要功能为封装成帧，透明传输和差错控制。

封装成帧：把网络层传下来的分组添加首部和尾部，用于标识帧的首部和尾部

透明传输：虽然使用了帧定界符来标识帧的开始和结尾，但是帧的数据部分也可能出现帧定界符，此时就需要加入转义字符，用户并不能察觉加入了转义字符，这个动作好像不存在一样，称为透明传输。

差错控制：帧中加入了FCS（帧检测序列），可以通过CRC(循环冗余检验)

物理层

在相邻的主机间传输数据比特流。

2.TCP和UDP的区别，什么场景使用TCP，什么场景使用UDP，那些协议使用到了TCP，UDP。

tcp和udp的区别

TCP（传输控制协议）：TCP是面向连接的，在传输数据前先要建立连接，数据传输结束后需要释放连接。TCP不能提供广播和多播，但是提供可靠的面向连接的数据传输服务。

UDP（用户数据报协议）：UDP是面向无连接的，因此UDP在传输时不需要先建立连接，同时接受方也不需要给出确认。

tcp和udp的使用场景

udp虽然不提供可靠传输，但是在语音和视频传输等强调性能的场景下，使用udp比较好，比如：QQ语音，直播等等。

tcp虽然提供可靠传输，但是它的开销比较大，适合使用在一些要求可靠性的场景下，比如文件传输，远程登陆等等。

那些协议使用到了tcp/udp

使用tcp的协议有：FTP（21号端口），Telnet(23)，SMTP（25）Http(80)

使用了udp的协议有：DNS(53)，TFTP(69)，SNMP等等

tcp需要几次握手，几次挥手，为什么？

tcp需要三次握手，四次挥手。

tcp为什么需要三次握手

三次握手的目的是建立可靠的通信，比如两军问题，A军和B军攻打C军，但是由于A，B两军没有驻扎在一起，他们又要交换作战计划，A把作战计划发给了B，但是A并不能确定B受到了作战计划，于是B需要给A发送确认，同时，B给A发送的确认，B也不知道A能不能受到，于是A还要给B发送确认。同时三次握手也可以避免服务器打开错误的连接，假如有一个连接请求滞留在网络中，客户端等待一个超时重传后又会发送一个连接请求，如果没有三次握手，服务器就会打开这两个连接。

三次握手的过程

首先服务器处于监听状态

第一次握手：客户端向服务器发送带有syn=1，ack=0，初始序号为x的请求连接报文。

第二次握手：服务器受到了客户端的请求报文。如果同意连接的会，会回传带有syn=1，ack=1，确认号为x+1，序号为y的报文。

第三次握手：客户端受到服务器的确认报文后，还要向服务器发出确认，确认号为y+1，序号为x+1.

B 收到 A 的确认后，连接建立。

tcp为什么需要四次挥手

服务端收到客户端的连接释放报文后，就进入close-wait状态，这个状态是为了服务端发送还没有发送完的数据，发送完后，服务端载发送连接释放报文。举个例子来说，假如A和B在通话，A告诉B说：我讲完了，B回复A说：我知道了，但是A不知道B有没有说完，所以要等B告诉A说：我讲完了。

tcp四次握手的过程

客户端发送连接释放报文，FIN = 1，seq = u。

服务端受到客户端的连接释放报文后发出确认，确认号为u+1，此时TCP属于半关闭状态，服务端可以向客户端发送数据，但是客户端不能向服务端发送数据。

当服务端不需要发送数据时，向客户端发送连接释放报文，FIN = 1，seq = w。

客户端收到服务端的连接释放报文，发送确认报文，服务端受到后释放连接。

TCP如何保证可靠传输

  1. 确认和重传：如果一个发送的报文段在一段时间内没有收到确认，就重传这个报文段。一个报文段从发送再到收到确认所经历的时间称为往返时间。
  2. 校验和：TCP报文头有校验和，用于检验数据在传输过程中是否发生了错误，如果收到的报文段的校验和有差错，则丢弃这个报文段。
  3. 流量控制：流量控制是为了控制发送方的发送速率，保证接受方来得及接收。
  4. 拥塞控制：如果网络出现拥塞，分组将会丢失，此时发送方会继续重传，从而导致网络更加拥塞，因此，当网络中出现拥塞时，应该控制发送方的速率。

滑动窗口

滑动窗口是缓存的一部分，发送方和接收方都维护了一个窗口，接收方通过 TCP 报文段中的窗口字段告诉发送方自己的窗口大小，发送方根据这个值和其它信息设置自己的窗口大小。

拥塞控制的四种算法

  1. 慢开始：由小到大逐渐增大发送窗口的大小，cwnd初始为1，每次增加一倍。
  2. 拥塞避免：当发送窗口到达一个临界值时，cwnd的大小每次增加1
  3. 快重传与快恢复：在接收端每次要对收到的最有一个有序的报文段进行确认，比如收到了M1，M2，M4，这是应当发送对M2的确认。在接收端如果收到了三个确认重复，就知道下一个报文段丢失，应当进行快重传，立即重传下一个报文段。

在浏览器中输入url，在敲回车会发生什么

总的来说会分为以下几个过程：

  1. DNS解析
  2. TCP连接
  3. 发送HTTP请求
  4. 服务器处理请求并返回HTTP报文
  5. 浏览器解析并渲染页面
  6. 结束连接

1. DNS解析

由与网络中寻址是使用的ip地址，而不是url，因此首先需要通过DNS找到域名对应的ip地址。DNS解析有分为递归查询和迭代查询。

递归查询的过程如下：

首先会在本地域名服务器中查询ip地址，在没有查到的情况下，本地域名服务器会向根域名服务器查询，如果根域名服务器还是查不到，本地域名服务器就会向顶级域名服务器查询。

当然为了优化dns查询，都会用到缓存，浏览器缓存，系统缓存，路由器缓存，等等。

2.TCP连接

由于http是建立在tcp上的，所以，建立http连接前会先建立tcp连接。

3.HTTP连接

建立TCP连接后客户端就可以向服务器发送http请求

4.服务器处理请求并返回HTTP报文

服务器收到http请求后就处理请求并发送Htttp响应报文

5.浏览器解析并渲染页面

6.结束连接

HTTP和HTTPS有什么区别

HTTP协议传输的数据都是未加密的，也就是明文传输，而HTTPS加入了SSL对HTTP协议传输的数据进行了加密，和身份认证。

  1. HTTPS需要到ca申请证书，HTTP不需要。
  2. HTTPS使用了SSL进行加密，而HTTP是明文传输
  3. HTTP和HTTPS使用的连接方式不同，端口也不一样
  4. http的连接很简单，是无状态的；HTTPS协议是由SSL+HTTP协议构建的可进行加密传输、身份认证的网络协议，比http协议安全。
